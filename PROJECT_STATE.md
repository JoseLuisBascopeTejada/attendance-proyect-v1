## 5. API Endpoints (as actually implemented)

Source: `backend/main.py` (direct read, 494 lines)

### GET `/`
- **Not prefixed with /api/** — root health endpoint
- Response: `{"service": "attendance-api", "version": "0.1.0"}`

### GET `/api/db-status`
- Request: none
- Success: `{"status": "ok", "db": "connected", "tables": ["attendance", "students"]}`

### POST `/api/register`
- Request: `multipart/form-data` — `file` (image), `name` (string)
- Validation chain:
  1. File size > MAX_FILE_SIZE_MB → **413** `{"detail": "File too large. Max 5MB allowed."}`
  2. Invalid image → **400** `{"detail": "Invalid image file."}`
  3. Dimensions > MAX_IMAGE_WIDTH × MAX_IMAGE_HEIGHT → **413** `{"detail": "Image dimensions too large. Max {w}x{h}px."}`
  4. No face detected → **400** `{"detail": "No face detected."}`
  5. Multiple faces → **400** `{"detail": "Multiple faces detected. Please use a photo with only one face."}`
  6. Confidence < FACE_CONFIDENCE → **400** `{"detail": "Face detected but image quality is too low. Please try again."}`
  7. Face area < MIN_FACE_AREA_RATIO of image → **400** `{"detail": "Face is too small in the frame. Please move closer to the camera."}`
  8. Cannot determine dimensions → **400** `{"detail": "Could not determine image dimensions."}`
- Success: `{"id": <int>, "name": <str>, "status": "registered"}`
- Side effects: inserts into `students` + `attendance` (type=check-in, method=individual)

### POST `/api/recognize`
- Request: `multipart/form-data` — `file` (image)
- Validation: same file-size/dimension checks as register
- No face in image → **404** (no body)
- Empty DB → **404** (no body)
- No match above threshold → **404** (no body)
- Success: `{"name": <str>, "distance": <float>, "student_id": <int>}` + optional `"duplicate": true`
- Side effects: inserts into `attendance` (type=check-in, method=individual) unless duplicate

### POST `/api/recognize-group`
- Request: `multipart/form-data` — `file` (image)
- Validation: same file-size/dimension checks
- No faces → `{"matches": []}`
- Empty DB → `{"matches": []}`
- Success: `{"matches": [{"name": <str>, "distance": <float>, "bbox": [x,y,w,h]}]}` + optional `"duplicate": true` per match
- Side effects: inserts into `attendance` (type=check-in, method=group) for each match

### GET `/api/attendance`
- Query params: `date_from`, `date_to`, `student_id`, `page` (default 1, min 1), `limit` (default 10, range 1–100)
- Success: `{"records": [{"id", "student_id", "name", "timestamp", "type", "method"}], "total": <int>, "page": <int>, "pages": <int>}`
- JOINs with `students` to include name

### GET `/api/students`
- Request: none
- Success: `{"students": [{"id", "name", "photo_path", "created_at"}]}`

### GET `/api/report/pdf`
- Query params: `date_from`, `date_to`
- Success: `application/pdf` with `Content-Disposition: attachment; filename=attendance_report.pdf`

### docker-compose.yml (as-is)

```yaml
services:
  backend:
    build: ./backend
    container_name: attendance_backend
    volumes:
      - ./backend:/app          # live code mount
      - ./data:/app/data        # persistent DB + face photos
    ports:
      - "8000:8000"
    environment:
      - PYTHONUNBUFFERED=1

  frontend:
    build: ./frontend
    container_name: attendance_frontend
    ports:
      - "3000:80"              # nginx serves on 80, mapped to 3000
    depends_on:
      - backend
```

### Key observations

- **`/app/data` is correctly volume-mounted** (`./data:/app/data`) — persists `attendance.db` and `known_faces/` across restarts.
- **`./backend:/app` is a live mount** — code changes on the host are reflected inside the container without rebuild. Only `requirements.txt` changes require `docker compose up --build`.
- **Backend `.env` is NOT mounted as a Docker `env_file`** — it's read by `load_dotenv()` in `main.py`. The `.env` file must exist at `backend/.env` on the host (or inside the container via the volume mount).
- **Frontend has no `env_file` or environment variables** — API URL is hardcoded in `client.ts`.
- **Frontend Dockerfile is multi-stage**: Node 22-alpine for build, nginx:alpine for serve. `ENV CI=true` prevents pnpm interactive prompts. `.dockerignore` excludes `node_modules` and `dist`.
- **Backend Dockerfile installs OpenCV system deps** (`libgl1`, `libglib2.0-0`), pins `opencv-python==4.11.0.86` via second pip install, and downloads `haarcascade_frontalface_default.xml` via urllib.
